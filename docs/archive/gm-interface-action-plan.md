# Plan d'action interface GM

## Principe directeur

Les onglets sont les grands espaces de travail du MJ.

Les panneaux sont les outils contextuels que le MJ ouvre, ferme, docke, deplace, reduit ou epingle pendant une session.

## Onglets GM cibles

- Campagne
- Preparation
- Session Live
- Personnages
- Journal
- Bibliotheque
- Parametres

## Campagne

Espace administratif et global.

Contenu :

- description de la campagne ;
- image de couverture ;
- systeme de jeu ;
- invitations ;
- membres ;
- roles ;
- statut ;
- prochaine session ;
- resume derniere session ;
- raccourcis vers personnages, scenes, notes et documents.

A eviter :

- carte ;
- combat ;
- jets temps reel ;
- panneaux de session.

## Preparation

Espace de preparation avant session.

Contenu :

- scenes ;
- cartes ;
- placement tokens ;
- PNJ ;
- rencontres ;
- notes secretes ;
- documents a reveler ;
- musiques ;
- ambiances ;
- quetes ;
- organisation par chapitre, lieu ou session.

## Session Live

Ecran principal pendant la partie.

Contenu :

- carte / VTT ;
- panneaux contextuels ;
- journal live compact ;
- modes de session.

Modes internes :

- Exploration ;
- Combat ;
- Roleplay ;
- Narration ;
- Preparation rapide ;
- Minimal.

Les modes ne sont pas des onglets. Ce sont des layouts de panneaux.

## Regle panneaux

Tous les panneaux doivent avoir :

- un seul header ;
- les memes boutons ;
- les memes etats ;
- la meme logique de dock ;
- la meme logique de sauvegarde ;
- la meme logique de reouverture.

Etats standards :

- ouvert ;
- ferme ;
- reduit ;
- docke ;
- flottant ;
- epingle ;
- verrouille ;
- redimensionnable ;
- repositionnable.
